                    ОS2. External course. Part 1

                    Nov 22, 2025 · 5 min read

                    Здесь представлено выполнение первого раздела курса “Системный администратор linux с нуля”.

                    Objective

                    Complete the first section of the external course “Linux System Administrator from Scratch”.

                    Task

                    Tasks from the first, second, and third modules, as well as tests.

                    Theoretical Introduction

                    • Module 1: Introduction to System Administration
                    • Module 2: Basic Linux Commands
                    • Module 3: Files and Directories

                    Practical Tasks Execution

                    Installing the htop utility.

                    Running htop and sorting processes by RAM usage.

                    Finding out how much free disk space is left.

                    Finding the subdirectories occupying the most space in the /var directory.

                    Reading the latest kernel-related messages from the system journal.

                    Checking the SSH status.

                    Getting a list of all active services.

                    Finding the configuration file for the SSH service. Opening it with a text editor, such as nano, and reviewing the settings specified there.

                    Viewing which logs are present in the system.

                    Viewing the last twenty entries in the system journal.

                    Finding out if “third-party” applications not managed by the package manager are used in the system.

                    Imagining that the Nginx web server is installed in the system and we need to reconfigure it. Going to the configuration directory, finding the file with the main settings, making a backup copy before changes, opening the file for editing, and restarting the service after changes.

                    Getting information about the processor.

                    Requesting data on memory usage. Paying attention to the fields: MemTotal, MemFree, Buffers, and others.

                    Viewing the list of files in the /dev directory.

                    Finding out the current directory. Moving to the home directory and getting a list of all files, including hidden ones. Viewing the last ten entries in the syslog system journal related to the systemd service.

                    Creating a ‘project’ directory, and inside it creating several subdirectories: logs, data, and backup.

                    Creating text files: error.log in the ~/project/logs/ directory and info.txt in the ~/project/data/ directory. Then verifying that the files have been created.

                    Copying the info.txt file from the ~/project/data directory to the ~/project/backup folder.

                    Moving the error.log file from the ~/project/logs/ directory to the ~/project/data/ folder.

                    Deleting the project folder along with all its contents.

                    Creating an archive_test.tar.gz archive using tar and gzip methods, placing three files inside it: file1.txt, file2.txt, file3.txt.

                    Extracting the archive_test.tar.gz archive into the ~/extracted directory.

                    Transferring the archive_test.tar.gz archive from the local computer to the server in the backups folder. Using the IP address 192.168.1.10 and the username ‘user’.

                    Creating a clean_logs.sh script that will archive all logs in the /var/log directory.

                    Making the script executable and running it.

                    Automating the script execution using cron. Configuring the clean_logs.sh script to run on the first day of each month at 3:30 AM.

                    Test Tasks Execution

                    Because the freedom defining free software is the freedom to study, modify, and improve the program.

                    Zabbix, as it is a specialized, powerful, and widely used monitoring system.

                    The key task of a system administrator is to ensure the stability and availability of the IT infrastructure.

                    Bash is the most common and standard command shell in most Linux distributions.

                    Man, when we need to get the most complete and structured reference information about a command, utility, or system call.

                    We elevate privileges when we need to perform an action affecting the entire system or other users.

                    We forgo the graphical interface on servers because it allows us to direct all computational resources to performing core service tasks, rather than rendering graphics.

                    We use SSH because this protocol provides us with an encrypted network connection for secure remote server management.

                    We distinguish these concepts as follows: A Terminal is a program that provides us with a text interface for entering commands and viewing results. A Shell is a command interpreter that runs inside the terminal, reads the commands we enter, executes them, and returns the result.

                    We know that the /etc directory is the standard location for storing global configuration files for the system and installed programs.

                    We refer to the /var/log directory because it is specifically intended for storing constantly growing files, which include logs.

                    We use the /opt directory to install third-party proprietary software or large software packages that come as self-contained builds and are not managed by the distribution’s package manager.

                    We view the /dev directory as a virtual filesystem where the Linux kernel represents all physical and virtual devices as special files.

                    We use the /proc virtual filesystem as a window into the system kernel.

                    We use the cat command to view the contents of virtual files in /proc and /dev because it works with any files, simply outputting their contents to the standard output.

                    We use the pwd command to precisely determine which directory we are currently in. This is a basic command for navigation and orientation in the filesystem. We use the -a flag with the ls command to display all files, including hidden ones whose names start with a dot. We use a relative path. Since we are already in the /etc directory, to navigate to its apache2 subdirectory, the command cd apache2 is sufficient.

                    We use the tilde symbol (~) as a shortcut that the shell automatically expands to the absolute path of the current user’s home directory. We build such a pipeline when we need to filter the detailed output of the ls -l command and keep only those lines containing the substring “ssh”.

                    We use the tar command with the -c, -z, and -f keys to create a compressed tarball archive from the specified folder. We use the tar command with the -x, -z, and -f keys to unpack a compressed tarball archive. We use the scp command with the -r key to securely copy an entire folder along with its contents via SSH to a remote server.

                    We use the chmod command with the +x key to give the current user the right to run the file as a program or script. We use the # symbol in Bash scripts and many other scripting languages to denote the start of a comment. We use this find command to search the /var/log directory for files modified more than 7 days ago and for each found file, execute the command to force its deletion.
